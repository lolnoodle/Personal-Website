## Why this project exists

Most developers slap together a portfolio in a weekend using a template and call it done. I wanted to do the opposite.

This project was built as a **deliberate learning experience** — a chance to level up my Next.js skills, get comfortable with TypeScript, learn animation with Framer Motion, and practice thinking like a professional developer rather than just someone who gets code to run.

I used AI (Claude) throughout this project — but not to write my code for me. I used it the way a junior dev would use a senior colleague: to get feedback on my architecture decisions, understand concepts before implementing them, review code I wrote myself, and ask questions I'd otherwise spend hours Googling. Every component in this repo was written by my own hands.

The difference between vibe coding and intentional development is understanding *why* something works — not just that it does. That was the standard I held myself to here.

## Tech stack

| Technology | Why I used it |
|---|---|

## How this was planned

This wasn't built by jumping straight into code. Before a single component was written there was:

- A full **sitemap** mapping every route and its content
- **Wireframes** for every page built in HTML
- A complete **design system** covering colors, typography, spacing, components and motion specs
- A **GitHub Projects board** with 42 issues across 8 epics tracking every feature from idea to done

That process is documented in the `/docs` folder if you want to see how the project was structured from the start.

## Design decisions

**Dark mode** — near-black base `#0a0a0a` with electric lime `#CAFF00` as the accent. High contrast, no compromise.

**Light mode** — white base with purple `#7B2FBE` as the accent. Inspired by Lil Uzi Vert's iconic hair. Yes really. Yes it's in the style guide.

**Bebas Neue** for all display headings — loud, compressed, commanding. The name should feel like a poster, not a business card.

**Custom crosshair cursor** — a small gaming detail that runs sitewide. The kind of thing you notice without knowing why the site feels different.
